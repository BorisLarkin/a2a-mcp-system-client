# A2A/MCP System Client

Клиентская часть интеллектуальной системы автоматизации обработки обращений в диспетчерских службах. Представляет собой набор компонентов для взаимодействия компаний-клиентов (диспетчерских) с центральным оркестратором.

## 📋 Состав репозитория

```
client/
├── local-db/ # Локальная база данных клиента
├── local-proxy/ # Прокси-сервер для безопасного соединения
├── scripts/ # Вспомогательные скрипты
├── telegram-bot/ # Telegram бот для приема обращений
├── web-dashboard/ # Веб-дашборд для администратора
├── .env.example # Пример конфигурации
├── docker-compose.yml # Docker Compose для локального развертывания
└── README.md
```

## 🏗️ Архитектура

Клиентская часть обеспечивает:
- **Прием обращений** через различные каналы (Telegram, Web)
- **Локальное хранение** данных и конфигураций
- **Безопасный прокси** для соединения с центральным оркестратором
- **Веб-интерфейс** для мониторинга и администрирования

## 🚀 Быстрый старт

### Предварительные требования
- Docker и Docker Compose
- Git

### Установка и запуск

```bash
# Клонирование репозитория
git clone https://github.com/your-org/a2a-mcp-system-client.git
cd a2a-mcp-system-client

# Настройка окружения
cp .env.example .env
# Отредактируйте .env под ваши параметры

# Запуск всех сервисов
docker-compose up -d

# Проверка статуса
docker-compose ps
```

### Настройка Telegram бота

Создайте бота через @BotFather

Получите токен

Добавьте токен в .env:

```env
TELEGRAM_BOT_TOKEN=your_bot_token_here
```

🔧 Конфигурация
Основные параметры в .env:

| Параметр | Описание | Значение по умолчанию |
| --- | --- | --- |
| `ORCHESTRATOR_URL` | URL центрального оркестратора | `http://orchestrator:8080` |
| `API_KEY` | Ключ доступа к платформе | `your_api_key` |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram бота | `-` |
| `DB_PASSWORD` | Пароль локальной БД | `postgres` |

## 📡 Взаимодействие с оркестратором

Клиент отправляет запросы на `/process-ticket`:

```json
POST /process-ticket
{
  "text": "У меня не работает интернет",
  "dispatcher_id": "uuid-диспетчерской",
  "metadata": {
    "channel": "telegram",
    "chat_id": 123456789
  }
}
```

## 📊 Мониторинг
Веб-дашборд доступен по адресу http://localhost:3000 (или настроенному порту) и предоставляет:

- Статистику обращений

- Историю тикетов

- Метрики производительности

- Статус подключения к оркестратору

## 🛠️ Разработка

### Запуск в режиме разработки

```bash
# Запуск отдельных сервисов
docker-compose up telegram-bot
docker-compose up web-dashboard
```

### Добавление нового канала

1. Создайте новый сервис в docker-compose.yml

2. Реализуйте отправку запросов на оркестратор

3. Добавьте обработку ответов
